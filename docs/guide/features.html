استخدم المناسب فقط

أفضل طريقة لاستخدام كل المميزات

INI Brain يوفر التوكينات عندما يمنع القراءة والتفسير المتكرر. لكنه قد يستهلك أكثر إذا طلبت سياقًا كبيرًا لسؤال صغير.

المسار اليومي الموصى به

Auto Brief مرة←Context مركز←التعديل←Impact + Tests←Save Memory
Prompt عملي«ابدأ بـauto brief، اقرأ السياق للمهمة فقط، عدّل أصغر عدد ملفات، افحص التأثير، شغّل الاختبارات، ثم احفظ القرار الدائم فقط.»

حقيقة توفير التوكينات

يوفر غالبًا عندما

  • المشروع متوسط أو كبير.
  • لديك جلسات متعددة.
  • توجد قرارات سابقة مهمة.
  • تستخدم Explain/Impact بدل قراءة ملفات كثيرة.
  • تحمّل المهارة المناسبة فقط.

قد يستهلك أكثر عندما

  • السؤال عام ولا يتعلق بالمشروع.
  • المشروع ملفان فقط.
  • تطلب Auto Brief وContext عدة مرات بلا داعٍ.
  • تفعّل Ultra ثم تضطر لإعادة السؤال بسبب إجابة غامضة.
  • تضع كل مهارة Always Apply؛ لذلك جعلنا Cursor عند الطلب.

للمقارنة المحلية استخدم:

ini_brain_compare_tokens {
  "baselineText": "النص الطويل قبل التحسين",
  "optimizedText": "النص الأقصر بعد التحسين"
}
العداد تقديريالنتيجة تقارن حجم النصين بنفس الخوارزمية. الاستهلاك الحقيقي يظهر في برنامج النموذج نفسه ويختلف حسب tokenizer.

ما هذه الملفات التي ظهرت في مشروعي؟

AGENTS.md

دستور قصير يقرأه Codex ووكلاء آخرون: أهم الملفات، القواعد، ومسار العمل.

.brain/compact_context.md

الملخص المركز الذي يوفر قراءة المشروع كاملًا كل مرة.

.brain/memories.json

ذاكرة محلية للقرارات والأخطاء والتفضيلات. لا تضع أسرارًا فيها.

.brain/upstream/

الملفات المنتقاة التي اختارها Smart Setup أو Upstream Apply، وليست clones.

.codex/.cline/.claude

مرايا مهارات بصيغة كل وكيل. المصدر الحقيقي يبقى `.brain`.

.specify/

مواصفات وخطط ومهام الميزات الكبيرة.

الأمان والخصوصية

  • الفحص وبناء الرسم والذاكرة محلي.
  • مفاتيح API داخل VS Code SecretStorage.
  • Auto Mode يعرض التغييرات ويطلب موافقة ويحفظ backup.
  • Smart Setup لا يثبت binaries ولا يستنسخ repos.
  • Memory Compact يبدأ Dry Run.
  • Apply Upstream يحتاج اختيارًا من المستخدم.
